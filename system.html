System Commands

File Management

pwd
print current working directory
ls [-la]
list directory (show permissions and hidden files)
cd [directory]
change directory
cd .. move 1 directory up
cd / move to the root directory
cd ~ move to current user’s home directory
mkdir [name]
creates a new directory
rm [name]
removes a file
rm -r [name] removes a directory
touch [name]
create a file
mv [filename] [full path of new location]
move a file or directory
mv [filename] [new name] used to rename a file or folder
cp [filename] [full path of new location]
copy a file or directory to a new location
cp -r [filename] [full path of new location] used to copy a directory
cat [file1] [file2] [file…]
displays file’s content, can display multiple files in 1 output
cat [file1] [file2] > [file3] combine multiple files
echo [text]
display text on screen
echo [text] >[file] add text to a file
clear
clear the terminal screen

File and Directory Permissions

permissions are a line of characters that look like this: -rwx-r–r–
first character in the line
- means file
d means directory
rest of characters
r readable
w write permissions to the file
x execute the file
the permissions are in groups of 3 which represent the owner/root, group, and others
setting permissions chmod - change mode and permissions
ugo = user, group, and others
ex: chmod u=rwx [file]
ex: chmod g-w [file]
octal mode
1: x
2: w
4: r
ex: chmod 761 [file]
notes
only the owner of the file and root are allowed to change permissions
write permissions are needed to create files or directories in a directory

File and Directory Ownership

chown [new owner] [file/folder]
change ownership of a file or folder
chgrp [new group] [username]
change group ownership
su [username]
change to another user account
(need that user’s password)

Search

grep [switches] [search input] [file or folder]
description: locate strings in a file
switches
-i: ignore case
-r: allows you to search a directory
-n display line that is greped
-w: only returns whole word (where a whole word matches the search input)
-v: inverted search, prints all lines that don’t include the search input
-A[# of lines]: number of additional lines printed after match term
-B[#number of lines]: before
repetition expressions
? The preceding item is optional and matched at most once.
* The preceding item will be matched zero or more times.
+ The preceding item will be matched one or more times.
{n} The preceding item is matched exactly n times.
{n,} The preceding item is matched n or more times.
{,m} The preceding item is matched at most m times. This is a GNU extension.
{n,m} The preceding item is matched at least n times, but not more than m times.
examples:
grep -i “ca.” file.txt
grep -i “pyt?h?o?n?” file.txt
would find python and py
notes
if you actually want to grep for a repetition expression, you have to use a backspace before it
ex: grep -r “hello/?” file.txt
